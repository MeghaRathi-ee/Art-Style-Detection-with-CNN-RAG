# 🎨 Art Style Detection using CNN and RAG

An end-to-end computer vision system that classifies paintings into art movements using a fine-tuned **EfficientNet-B0** CNN, then generates rich historical and biographical context using **Retrieval-Augmented Generation (RAG)** with a fully local large language model.

---

## Demo

Upload any painting → get the predicted art style + a detailed historical explanation generated by a local LLM.

![Streamlit UI](images/demo.png)

---

## Features

- **27-class art style classification** using EfficientNet-B0 fine-tuned on 81,444 WikiArt paintings
- **Top-5 predictions** with confidence scores
- **RAG pipeline** that retrieves relevant art history knowledge from a vector store and generates contextual explanations
- **Fully local inference** — no API keys, no cloud, no internet required after setup
- **Interactive Streamlit web app** with drag-and-drop image upload

## System Architecture

```
Input Image
    │
    ▼
┌─────────────────────────────────────────────┐
│              Phase 1 — CNN                  │
│                                             │
│  Preprocessing → EfficientNet-B0 →          │
│  Custom Head (Drop→Lin512→GELU→Lin27)        │
│                                             │
│  Output: Style label + confidence scores    │
└─────────────────────────────────────────────┘
    │
    │ style label
    ▼
┌─────────────────────────────────────────────┐
│           Phase 2 — RAG Pipeline            │
│                                             │
│  SBERT Embedding → ChromaDB Query →         │
│  Top-2 chunks retrieved →                   │
│  llama3.2 (Ollama) generates explanation    │
│                                             │
│  Output: Historical context (3-4 paragraphs)│
└─────────────────────────────────────────────┘
    │
    ▼
Streamlit Web Application
```

## Setup and Installation

### Prerequisites
- Python 3.11
- [Ollama](https://ollama.com) installed on your machine
- (For training) A CUDA GPU or access to a GPU cluster

### 1. Clone the repository
```bash
git clone https://github.com/YOUR_USERNAME/art-style-detection.git
cd art-style-detection
```

### 2. Create a virtual environment
```bash
python3.11 -m venv venv
source venv/bin/activate
```

### 3. Install dependencies
```bash
# PyTorch (CPU — for Mac Intel)
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu

# All other dependencies
pip install -r requirements.txt
```

### 4. Pull the LLM model
```bash
ollama pull llama3.2
```

### 5. Start Ollama (keep this running in a separate terminal tab)
```bash
ollama serve
```

### 6. Run the Streamlit app
```bash
streamlit run app.py
```

Open your browser at `http://localhost:8501`, upload any painting, and get the result.

## Training from Scratch

### 1. Download the WikiArt dataset
```bash
pip install kaggle
# Place your kaggle.json in ~/.kaggle/
kaggle datasets download -d steubk/wikiart
unzip wikiart.zip -d data/wikiart
```

### 2. Train the model
```bash
# On a GPU cluster
python train.py --data_dir data/wikiart --epochs 20 --batch_size 64

# On CPU (slow — use small epochs for testing)
python train.py --data_dir data/wikiart --epochs 3 --batch_size 16
```

Training uses a two-phase strategy:
- **Phase 1 (epochs 1–5):** Backbone frozen, only the classifier head is trained
- **Phase 2 (epochs 6–20):** All layers fine-tuned with a low learning rate

### 3. Run inference
```bash
python main.py --image path/to/painting.jpg --model_path art_classifier.pth
```

## How RAG Works

1. The CNN produces a style label (e.g. "Post_Impressionism")
2. The label is embedded using `all-MiniLM-L6-v2`
3. ChromaDB finds the most relevant knowledge chunks using cosine similarity
4. The top-2 chunks are passed as context to `llama3.2`
5. The LLM generates a 3–4 paragraph response covering:
   - Historical and cultural context of the movement
   - Visual characteristics the CNN detected
   - Key artists and their celebrated works
   - Legacy and influence on later art
